# Лабораторная работа №23. Интерактивные веб-приложения на JavaScript

## Описание игры

Крестики-нолики — игра для двух игроков на поле 3×3. Игроки по очереди ставят свои символы (первый игрок — X, второй — O) в пустые клетки. Цель игры — первым выстроить непрерывную линию из трёх своих символов по горизонтали, вертикали или диагонали. Если все 9 клеток заполнены, но ни один игрок не выстроил линию, объявляется ничья.

### Правила игры

### Логика победы/поражения

Победа наступает, когда один из игроков заполняет три клетки своим символом в одной из восьми возможных выигрышных комбинаций. Поражение как таковое отсутствует — игра заканчивается либо победой одного из игроков, либо ничьей.

### Особенности реализации

- Игра реализована на JavaScript
- Интерфейс обновляется динамически через DOM-элементы
- Состояние игры хранится в массиве и синхронизируется с визуальным отображением
- Реализована защита от повторного клика по занятой клетке
- После окончания игры все клики игнорируются до перезапуска

## Игровая механика:

### Как определяется победитель?

Победитель определяется путём проверки всех выигрышных комбинаций после каждого хода. В коде создан массив winConditions, содержащий 8 комбинаций индексов клеток (3 горизонтали, 3 вертикали, 2 диагонали). Функция checkResult() проверяет, заполнены ли все три клетки любой комбинации одинаковыми символами (и не пустые). При обнаружении такой комбинации игра завершается, и отображается сообщение о победителе.

### Как обрабатываются клики?

При клике на клетку срабатывает функция `handleCellClick(event)`. Она получает индекс кликнутой клетки через data-index. Затем проверяется, свободна ли клетка и активна ли игра. Если условия соблюдены, символ текущего игрока записывается в массив `board` и отображается в клетке. После этого вызывается функция `checkResult()` для проверки состояния игры.

### Как работает перезапуск?
 
Функция `restartGame()` сбрасывает все игровые переменные: текущий игрок становится "X", массив board очищается, флаг `gameActive` устанавливается в true, а текст статуса возвращается к начальному сообщению "Ход игрока: X".

## Структура данных:

### Объяснить массив board

Массив board представляет игровое поле и содержит 9 элементов, каждый из которых соответствует одной клетке (индексы от 0 до 8). Индексы расположены слева направо, сверху вниз. Каждый элемент массива хранит либо пустую строку "", либо символ "X" или "O". 

### Объяснить условия победы

Условия победы хранятся в массиве `winConditions`, который содержит 8 вложенных массивов. Каждый вложенный массив содержит три индекса, образующих выигрышную линию. При проверке победы программа последовательно проходит по всем комбинациям и сравнивает значения в массиве board по указанным индексам.

### Как хранится состояние игры?

Состояние игры хранится в трёх основных переменных:

- `board` — массив с текущим заполнением клеток
- `currentPlayer` — символ текущего игрока ("X" или "O")
- `gameActive` — логический флаг, указывающий, продолжается ли игра или завершена

## Алгоритмы:

### Алгоритм проверки победы

1. Установка флага `roundWon = false`
2. Проход циклом по всем комбинациям из winConditions
3. Для каждой комбинации записывается три индекса [a, b, c]
4. Проверка: значение в `board[a]` не пустое и равно значению в `board[b]` и равно значению в `board[c]`
5. Если условие выполнилось: `roundWon = true` и остановка цикла
6. Если `roundWon === true`:
    - Сообщение о победе
    - `gameActive = false`
    - Завершение выполнения функции

### Алгоритм определения ничьи

1. После проверки победы (если победы нет) проверка массива `board` на наличие пустых строк
2. Метод `board.includes("")` — возвращает true, если есть хотя бы одна пустая клетка
3. Если пустых клеток нет `(!board.includes("") === true)`:
    - Сообщение "Ничья!"
    - `gameActive = false`
    - Завершение выполнения функции

### Алгоритм смены игрока

1. Если победа не обнаружена и ничья не наступила, игра продолжается
2. Смена текущего игрока с помощью:
    - Если `currentPlayer === "X"`, то новое значение "O"
    - Если `currentPlayer === "O"`, то новое значение "X"
3. Обновление текста статуса с сообщением о том, чей теперь ход
